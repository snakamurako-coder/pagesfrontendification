# 漢字ニガテ特訓（4モード＋スコープ）設計プラン

## 実装状況スナップショット（ローカルコード観測ベース）

設計ドキュメントの YAML が「completed」になっていたが、現時点のコードベースでは **弱みシグナル系（`append_kanji_weak_signals` / `__kanjiWeak` / iframe の boolean 契約など）は未実装**で、`コード.js` にも該当シンボルが見当たらない。

一方で次は **既に存在**する。

- **漢字練習カタログのスコープ UI**: `index.html` の `kp-book-select` / `kp-sheet-select` / `kp-set-select`（ブック→シート→セット）
- **通常の「ニガテ特訓」ボーナス（英語クイズ側の設定）**: `setting-play-mode` の review と `handleSaveLearningSession` の `isReviewMode` 倍率
- **漢字クイズ・練習のポイント付与の一次経路**: `save_learning_session`（漢字は `learningCategory: kanji` + `challengeType: score` 分岐）

## ビルドバックログ（未完を実装へ落とす）

目的は「設計どおり、薄いシグナルを溜め、スコープ付きで復習出題できる」こと。**細かいストロークログは保存しない**。

### M1 — iframe 契約（最小）

- `KP_HTML` 内で `evaluateKanji` 完了後に親へ `postMessage`（例：`type: "kpKanjiEvalSummary"`）
- フィールドは設計の boolean のみ（`hasStrokeOrderIssue`, `brushEndingAllOk` or `hasBrushEndingIssue`, `referenceStrokeCount?`, `handScore`, `passedThreshold`, `kanjiChar`）
- 親の `patchKanjiFrameForQuizPostMessage` で **`kanjiQuizScored`（点数だけ）とは別チャネル**で受ける（練習保存との干渉を避ける）

### M2 — GAS：弱みシグナル保存

- `コード.js` に `handleAppendKanjiWeakSignals`（命名は統一）を追加し `doPost` 分岐
- `users.historyJson.__kanjiWeak` にマージ（上限・ガベージコレクションは GAS 側で固定）
- **ポイント付与とは切り離す**（必要なら同日バッチや軽いデバウンスのみ）

### M3 — GAS：復習プラン API

- `get_kanji_weak_review_plan`（または既存 `get_kanji_quiz_questions` のフィルタ版）
- 入力：スコープ（modeId/unit/set または「全部」）、軸、上位 N、除外ルール
- 出力：問題 ID / kanji のリスト（クライアントは既存プレイへ渡すだけ）

### M4 — フロント：4軸モードと終了メッセージ

- 既存の漢字クイズプレイに「特訓（弱み）」入口を追加（スコープは `kp-*` と整合させる）
- セッション終了時は設計どおり **集約 boolean メッセージのみ**（ストローク単位 UI は作らない）

受け入れ条件（最小）

- 手書き採点後に **boolean シグナルが保存**される（個々のストローク列は保存されない）
- スコープを変えると **復習候補が変わる**
- 既存の通常学習・ポイント付与が壊れない（後方互換）

## 方針（収集粒度・フィードバック）

- **採点ソース**: 引き続き `KP_HTML` / iframe 内の採点ロジックを正とする（代理だけの MVP は取らない方針は維持してよいが、**細かい診断ログは不要**）。
- **① 書き順**: 「**間違い指摘があったか**」程度の **boolean**（例 `hasStrokeOrderIssue`）のみ収集。**何画目・どの順位がずれたか**は保存しない。
- **③ とめ・はね・はらい**: 「**すべてクリアできたか**」または「**いずれか未達があったか**」の **boolean**（例 `brushEndingAllOk` / `hasBrushEndingIssue`）のみ。**何画目が未達か**は保存しない。
- **② 画数**: 必要なら参照画数や画数クイズの正誤など**既に1値で表せるもの**に限定（詳細ストローク列は不要）。
- **ニガテ特訓の UI フィードバック**: セッション／セット単位で **「書き順の間摘はなかった」「とめはねはらいはすべてクリア」** のような**総合メッセージ**に留める。**細かい懸賞・逐条の指摘 UI は作らない**。

## 現状との接点

- **学習履歴**: 管理ブック `users` の `historyJson`（列6）。漢字ドリルは [`handleSaveLearningSession`](c:\Users\enhan\コード開発\Projects\pagesfrontendification\コード.js) の `questionId` / `questionCorrect` で履歴が積まれる。
- **ニガテ判定の先例**: [`checkIsNigate`](c:\Users\enhan\コード開発\Projects\pagesfrontendification\index.html) と `アプリ設定` の「苦手_*」。漢字軸は **boolean 失敗回数** で弱み度を上げるだけで足りる。

## 目標データモデル（アルゴリズムの核）

`historyJson.__kanjiWeak`（名前空間は実装時に確定）

各レコードに必須:

- `modeId`, `unitName`, `setId`, `kanji`
- 軸ごとに **カウンタ**（例 `strokeOrderFailCount`）または **直近 K 件の boolean サマリ**（タイムスタンプ + `hasStrokeOrderIssue` のみ）を保持。**配列で全ストロークを溜めない**。

集約例:

- ①: `hasStrokeOrderIssue === true` のイベントで `w_strokeOrder` を増分。
- ③: `brushEndingAllOk === false` で `w_brush` を増分；`true` で軽減またはノイズ扱いは設計で調整。

## iframe から返すデータ（最小契約）

採点確定後 **1 回／問**、`postMessage` 例（フィールド名は実装で固定）:

| 軸 | 送るもの（例） | 送らないもの |
|----|----------------|--------------|
| ① 書き順 | `hasStrokeOrderIssue: boolean`（採点ロジックが「筆順に問題あり」と判定したとき true） | 逸脱インデックス、期待順と実順の列、画番号 |
| ③ とめ・はね・はらい | `brushEndingAllOk: boolean`（全ストロークの終筆判定が閾値を満たす） | 画ごとの expected/observed 列 |
| ② 画数 | 任意: `referenceStrokeCount: number` と画数クイズ正誤、または省略可 | ストロークごとの詳細 |
| 共通 | `handScore`, `passedThreshold` | — |

iframe 内で「間摘あり／なし」「終筆すべて OK」が **1 bit で決められない**場合のみ、実装時に最小限の内部集約（例: 内部でループして最終 boolean だけ算出）し、**親には boolean のみ露出**。

**親ページ**: [`patchKanjiFrameForQuizPostMessage`](c:\Users\enhan\コード開発\Projects\pagesfrontendification\index.html) で受信 → `append_kanji_weak_signals`。

## 出題・特訓フィードバック

- 弱み候補は **boolean 失敗の積み上げ**でランキング。
- 特訓終了画面: **集約文のみ**（書き順問題なし／終筆すべて OK 等）。**細かい懸賞・ストローク単位の演出はスコープ外**。

## 妥当性への助言

- **メリット**: プライバシー・payload・実装コストが下がる。ユーザー向け説明もシンプル。
- **注意**: iframe 側は「boolean を出すための**内部**判定」は依然必要。親に渡すのは結果の要約のみ。
- **画数**: 教育字体との差の注記は引き続き有効。

## 実装タスク（推奨順）

2. **親**受信 → **append_kanji_weak_signals** → `__kanjiWeak` マージ。
3. **スキーマ**確定（カウンタ + 直近数件の boolean のみ）。
4. **スコープ UI** + **復習プラン API**（任意）。
5. **4 モード出題** + **特訓終了の集約フィードバック**のみ。
